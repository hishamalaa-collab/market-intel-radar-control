# Operating Principles

## 1. Documents Are Inputs

Documents are not the final product. They are raw material for topics, insight, reusable outputs, and commercial thinking.

## 2. Instructions Need Version Control

If a rule, role, workflow, quality gate, or prompt is important, it belongs in GitHub.

## 3. Drive Holds Files, Not Control Logic

Google Drive is for source material, working files, client materials, exports, and assets. It is not the source of truth for operating rules.

## 4. No Reinvention Without Permission

Approved copy, approved structure, approved brand rules, and approved workflows must be preserved unless the user explicitly asks for a change.

## 5. Small Changes Beat Big Drifts

Every update should be small, named, and reviewable.

## 6. Every Workflow Needs a Quality Gate

A process is not complete until it has a review/checklist mechanism.
